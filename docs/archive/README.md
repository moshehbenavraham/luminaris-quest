## üìú Document Retention Policy

### Why We Archive

Instead of deleting old documentation, we archive it because:

1. **Historical Value:** Shows how the project evolved
2. **Problem-Solving Reference:** Past solutions may inform future decisions
3. **Audit Trail:** Maintains record of decisions and changes
4. **Learning Resource:** Examples of documentation and approaches
5. **Compliance:** Some organizations require document retention

### What Gets Archived

Documents are archived when they:

- ‚úÖ Describe completed one-time tasks (bug fixes, migrations)
- ‚úÖ Are superseded by newer documentation
- ‚úÖ Contain duplicate information now consolidated elsewhere
- ‚úÖ Are specific to temporary situations or platforms
- ‚úÖ Are marketing materials rather than technical docs
- ‚úÖ Belong in code comments rather than standalone files

### What Stays Active

Documents remain active when they:

- ‚úÖ Describe current features and architecture
- ‚úÖ Provide essential guides for users or developers
- ‚úÖ Document current APIs and interfaces
- ‚úÖ Offer troubleshooting for current issues
- ‚úÖ Guide ongoing development and contributions

## üìù Archive Maintenance

### Regular Reviews

The archive is reviewed periodically to:

- Ensure archived docs are still relevant to keep
- Remove truly obsolete documents after sufficient time
- Update this README with accurate status information
- Verify links and metadata

### Contributing to Archive

If you find documentation that should be archived:

1. **Verify it's truly obsolete:** Check with maintainers
2. **Extract any unique value:** Merge useful info into active docs
3. **Move to archive:** Use git mv to preserve history
4. **Update this README:** Add entry with proper metadata
5. **Update links:** Ensure no broken links in active docs

See [Contributing Guide](../contributing/index.md) for process details.
